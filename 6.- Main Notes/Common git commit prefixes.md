# Common git commit prefixes


The Conventional Commits specification outlines several common prefixes:

- **`feat:`**: Indicates a new feature. Example:Â `feat: add user authentication functionality`
- **`fix:`**: Signifies a bug fix.
- **`docs:`**: For documentation changes.
- **`style:`**: Used for code style updates that don't alter the code's meaning.
- **`refactor:`**: Denotes code changes that are neither new features nor bug fixes.
- **`perf:`**: Represents code changes that improve performance.
- **`test:`**: For adding or correcting tests.
- **`chore:`**: Indicates routine maintenance, such as dependency updates.
- **`build:`**: Used for changes affecting the build system or external dependencies.
- **`ci:`**: Signifies changes to CI configuration or scripts.
- **`revert:`**: Denotes the reversion of a previous commit.
